# Хранение регистраций турнира — переиспользуем таблицу такси/развоза

Ничего нового настраивать не нужно. Заявки турнира уходят в **уже существующую** таблицу
«Реестр счетов СПиР (тест)», лист **«Заявки портала»** — тот же Apps Script-вебхук, что у развоза
(`SHUTTLE_WEBHOOK_URL`). Такси-заявок там сейчас нет, таблица свободна.

## Что нужно сделать (1 шаг)
Вставить URL вебхука в сайт. Открой `pizza-battle/index.html`, найди:

```javascript
const WEBHOOK_URL = "PASTE_SHUTTLE_WEBHOOK_URL_HERE";
```

и подставь свой `SHUTTLE_WEBHOOK_URL` (вида `https://script.google.com/macros/s/…/exec`).
Где взять: **Netlify → сайт baden-inside → Site settings → Environment variables → `SHUTTLE_WEBHOOK_URL`**
(или в самой таблице: Расширения → Apps Script → Развернуть → Управление развёртываниями → URL веб-приложения).

Больше ничего — Apps Script менять не надо.

## Как заявки лягут в таблицу (лист «Заявки портала»)
| Колонка | Что попадёт |
|---|---|
| Дата заявки | автоматически (время отправки) |
| ФИО | Имя и фамилия участника |
| **Подразделение** | **🍕 ТУРНИР** ← метка, по ней фильтруем турнирные строки |
| Дата поездки | выбранная дата отборочного этапа |
| Адрес высадки | Город участника |
| Телефон | Телефон участника |
| Комментарий | «Термальный батл · этап … · 18+: да · согласие ПДн: да» |
| Статус | NEW |

## Как скачать список зарегистрированных
1. Открой таблицу → лист **«Заявки портала»**.
2. Поставь фильтр по колонке **«Подразделение» = 🍕 ТУРНИР** (только турнирные заявки, без будущих такси).
3. **Файл → Скачать → Excel (.xlsx)** или CSV. Случайный выбор: колонка `=RAND()` → сортировка → первые N.

## (Опционально) отдельный лист «Турнир»
Если позже захочешь, чтобы турнир писался в отдельную вкладку, а не в «Заявки портала» —
добавь в Apps Script в начало `doPost` роутинг по метке. Скажи — дам готовый сниппет.
